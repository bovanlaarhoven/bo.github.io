Scripting Guide

Getting Started with Scripts

Learn how to create and execute powerful scripts with Bosploit's Lua scripting environment.

Basic Script Structure

All Bosploit scripts are written in Lua and follow a simple structure:

-- Simple Hello World script
log("Hello from Bosploit!")

-- Get player information
local player = getPlayer()
log("Player name: " .. player.name)
log("Player position: " .. player.x .. ", " .. player.y .. ", " .. player.z)

Script Examples

Auto Mining Script

-- Auto mining script that mines in a straight line
local distance = 50
local player = getPlayer()

for i = 1, distance do
    -- Move forward
    player.moveForward()
    wait(0.1)
    
    -- Mine the block in front
    local frontBlock = getBlock(player.x + 1, player.y, player.z)
    if frontBlock.type ~= "air" then
        mineBlock(player.x + 1, player.y, player.z)
        wait(0.5)
    end
end

log("Mining complete!")

Auto Combat Script

-- Auto combat script that attacks nearby enemies
while true do
    local enemies = getNearbyEntities("hostile", 10)
    
    if #enemies > 0 then
        local target = enemies[1]
        aimAt(target.x, target.y + 1, target.z)
        attack(target)
        log("Attacking: " .. target.type)
        wait(0.5)
    else
        wait(1) -- No enemies, wait longer
    end
end

Building Script

-- Build a simple house
local player = getPlayer()
local startX, startY, startZ = player.x, player.y, player.z

-- Build walls
for x = 0, 10 do
    for z = 0, 10 do
        for y = 0, 3 do
            if x == 0 or x == 10 or z == 0 or z == 10 then
                setBlock(startX + x, startY + y, startZ + z, "cobblestone")
            end
        end
    end
end

-- Build roof
for x = 0, 10 do
    for z = 0, 10 do
        setBlock(startX + x, startY + 4, startZ + z, "oak_planks")
    end
end

log("House built successfully!")

Best Practices

  • Use wait() functions: Prevent your scripts from running too fast and causing lag
  • Error handling: Use if statements to check for valid objects before using them
  • Modular code: Break complex scripts into smaller functions
  • Comments: Document your code with comments for future reference
  • Testing: Test scripts in single-player before using on servers

Advanced Features

Event Handling

-- Register event listeners
onPlayerDamage(function(damage)
    log("Player took " .. damage .. " damage!")
    if getPlayer().health < 5 then
        useItem("golden_apple")
    end
end)

onBlockBreak(function(x, y, z, blockType)
    log("Broke " .. blockType .. " at " .. x .. ", " .. y .. ", " .. z)
end)

GUI Integration

-- Create a simple GUI
local gui = createGUI("My Script", 300, 200)
gui:addButton("Start Mining", function()
    startMining()
end)
gui:addSlider("Speed", 1, 10, 5, function(value)
    miningSpeed = value
end)
gui:show()

Script Management

Tips for organizing and managing your scripts:

  • Save scripts in the scripts/ folder
  • Use descriptive filenames like auto_farm.lua
  • Create folders for different script categories
  • Backup important scripts regularly
  • Share scripts with the community through GitHub
